# Data Ingestion

This repo should commit the ingestion code and schemas, not the private source
data or generated ground truth tables.

## Privacy Boundary

Keep these local unless there is an explicit release review:

- raw source files under `<path-to-private-data-directory>`
- extracted tables under `data/processed/`
- generated task bundles under `data/tasks/`
- hidden answer files under `data/answers/`
- model run outputs under `runs/`

The `.gitignore` is configured for that boundary. Treat `data/processed/` as the
local private database for benchmark construction.

## Install

Use `uv`:

```bash
uv sync
```

## Populate In Vitro Tables

The in vitro mastersheet is converted into normalized CSV tables:

```bash
uv run capablebench ingest "<path-to-invitro-mastersheet.xlsx>"
```

Outputs:

```text
data/processed/peptides.csv
data/processed/invitro_assays.csv
data/processed/plate_qc.csv
data/processed/ingest_summary.json
```

`peptides.csv` has one row per anonymized peptide ID. `invitro_assays.csv` has
one row per peptide-assay-condition record. `plate_qc.csv` contains internal
plate QC records.

## Populate In Vivo Tables

The mouse data folder contains Excel workbooks plus JSON exports. Extract both
with:

```bash
uv run capablebench extract-invivo "<path-to-mouse-data-directory>"
```

Outputs:

```text
data/processed/invivo_measurements_long.csv
data/processed/invivo_studies_inventory.csv
data/processed/invivo_json_exports_raw_measurements_long.csv
data/processed/invivo_json_exports_cage_in_rack.csv
data/processed/invivo_json_exports_analysis_bars.csv
data/processed/invivo_json_exports_analysis_scatter.csv
data/processed/invivo_json_exports_analysis_significance.csv
data/processed/invivo_extract_summary.json
```

`invivo_measurements_long.csv` contains hourly Excel-derived measurements in tidy
format. `invivo_json_exports_raw_measurements_long.csv` contains 10-minute raw
JSON measurements. The `analysis_*` tables contain derived group/window
summaries and significance calls from analysis exports.

The existing high-resolution aMCH summary is not generated by this command, but
is a useful source to copy or integrate later:

```text
<path-to-mouse-data-directory>/aMCH_highres_hourly_analysis_20260403_codex/hourly_stats.csv
```

## Inspect The Local Database

Basic checks:

```bash
uv run capablebench list-tasks
cat data/processed/ingest_summary.json
cat data/processed/invivo_extract_summary.json
sed -n '1,5p' data/processed/invitro_assays.csv
sed -n '1,5p' data/processed/invivo_measurements_long.csv
```

Count studies and measurements:

```bash
python3 - <<'PY'
import csv, collections
for path in [
    "data/processed/invivo_measurements_long.csv",
    "data/processed/invivo_json_exports_raw_measurements_long.csv",
]:
    studies = collections.Counter()
    measurements = collections.Counter()
    with open(path, newline="") as f:
        for row in csv.DictReader(f):
            studies[row.get("study_id") or row.get("study_name")] += 1
            measurements[row["measurement"]] += 1
    print(path)
    print("studies:", len(studies))
    print("measurements:", len(measurements))
    print()
PY
```

## Populate Benchmark Tasks

Task bundles should be curated from linked processed tables. Put task
directories under `data/tasks/<task_id>/` and hidden answer YAML files
under `data/answers/<task_id>.yaml`.

Each task set should also include `data/tasks/problems.csv` so the runner can
discover tasks with `list-tasks`.

Build the current private benchmark directly from the processed private tables:

```bash
uv run capablebench curate-pilot --clean
uv run capablebench validate
```

The curation command currently creates:

- candidate prioritization tasks from linked in vitro and held-out in vivo
  outcome summaries,
- hit-prediction tasks from held-out significance calls,
- next-experiment tasks with expert utility labels,
- QC/failure-diagnosis tasks,
- mechanistic cross-family hypothesis tasks,
- foundation-model-triage tasks with model-output evidence,
- end-to-end drug discovery program planning tasks.

BioDiscoveryBench treats every new private dataset as an evidence layer. Current
in vitro and in vivo layers support candidate ranking, hit prediction, failure
diagnosis, and next-experiment tasks. Future omics, structure,
developability, model-output, and decision-history layers should be added as
task-local files with provenance metadata instead of changing the runner
contract.

## Run A Task

```bash
uv run capablebench run TASK_ID \
  --agent-command 'codex exec --json --cd {task_dir} "$(cat {prompt_file})"'
```

For a full local suite:

```bash
uv run capablebench run-suite \
  --agent-command 'codex exec --json --cd {task_dir} "$(cat {prompt_file})"' \
  --limit 10
uv run capablebench summarize
```

## Rebuild From Scratch

Generated processed tables are reproducible. To rebuild the local private
database:

```bash
uv run capablebench ingest "<path-to-invitro-mastersheet.xlsx>"
uv run capablebench extract-invivo "<path-to-mouse-data-directory>"
```
