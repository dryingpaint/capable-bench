# Calibration

BioDiscoveryBench should be calibrated so current frontier coding agents do not
saturate the benchmark. The target for the first serious pilot is less than
`0.60` mean task-normalized score for both Codex and Claude on the held-out task
set, while still allowing domain experts to verify that high-scoring answers are
scientifically useful.

## Required Inputs

Calibration requires real curated tasks under `data/tasks/` and hidden labels
under `data/answers/`. The repository currently keeps those directories
gitignored because benchmark packets may contain private scientific data.

Before running calibration, verify:

```bash
uv run capablebench list-tasks
```

## Local Codex Run

```bash
uv run capablebench run-suite \
  --agent-command 'codex exec --json --cd {task_dir} "$(cat {prompt_file})"' \
  --timeout-seconds 1800
uv run capablebench summarize
```

## Local Claude Run

```bash
uv run capablebench run-suite \
  --agent-command 'claude -p --output-format stream-json --verbose "$(cat {prompt_file})"' \
  --timeout-seconds 1800
uv run capablebench summarize
```

## Modal Runs

Use Modal for larger sweeps once the Modal CLI is available in the active Python
environment and authenticated:

```bash
export OPENAI_API_KEY=...
modal setup
uv run capablebench run-suite \
  --remote modal \
  --agent-command 'codex exec --json --cd {task_dir} "$(cat {prompt_file})"' \
  --timeout-seconds 1800
```

Run the Claude command the same way if the remote image has a configured Claude
CLI and credentials.

## Saturation Gate

After each calibration run, inspect `runs/latest_suite_summary.json`.

The pilot is considered too easy if either agent reaches:

- `mean_score >= 0.60` across all graded tasks,
- `mean_precision_at_k >= 0.60` on candidate prioritization tasks,
- `exact_match_rate >= 0.60` on hit prediction tasks,
- expert review finds that high scores can be achieved by sorting one obvious
  column or parroting the prompt.

If the suite is too easy, add harder cases with conflicting evidence, missing
modalities, noisy assay panels, negative controls, and foundation-model outputs
that are useful but imperfect.

Expert-rubric tasks use an `auto_score_cap` in their hidden answer YAML. This
keeps automated keyword coverage useful for smoke testing without treating it as
equivalent to expert scientific review. Final benchmark reports should separate
automated scores from expert-adjudicated rubric scores.

## Current Local Calibration

The current private pilot generated by `uv run capablebench curate-pilot --clean`
contains 52 tasks and passes `uv run capablebench audit-quality`.

Latest local calibration artifacts are summarized in `runs/calibration_summary.json`:

- Codex stratified 11-task slice: `mean_score = 0.4970`,
- Claude stratified 11-task slice: `mean_score = 0.5879`.

Both agent slices are below the `<0.60` saturation target under automated
scoring. Expert-review tasks still need human adjudication before publication.

## Reporting

Report calibration by task family, not only as one aggregate. A good report
should include:

- number of tasks and task families attempted,
- Codex and Claude mean task-normalized scores,
- per-family ranking, label, option-ranking, and rubric scores,
- expert-review notes for creative scientific tasks,
- examples of common failure modes,
- whether the suite passed the `<0.60` saturation target.
